# 术语表

## 一、文档目的

这份术语表用来统一项目里的常见说法，避免同一个东西在不同文档、不同模块里叫法不一致。

它主要解决这些问题：

- 前后端对同一概念理解不同
- 数据字段名和业务叫法对不上
- 英文名、中文名、代码名混用
- 一个词在不同上下文里含义不一样

## 二、使用规则

### 2.1 优先使用统一叫法

写文档、写接口说明、讨论需求时，优先使用这份术语表里的推荐叫法。

### 2.2 字段名和展示名分开

例如：

- 文档展示可以写“学习进度”
- 数据库字段可以是 `learning_progress`

两者可以不同，但要知道它们是同一个概念。

### 2.3 遇到歧义词要补上下文

比如 `level` 这个词，在项目里至少可能表示：

- JLPT 等级
- SRS 等级
- 游戏关卡等级

讨论时不要只说 “level”，要说清楚是哪一种。

## 十二、容易混淆的词

### 12.1 level

这个词最容易混：

- `JLPT level`：N5、N4、N3、N2、N1
- `SRS level`：记忆等级
- `game level`：游戏关卡

建议在文档和讨论里尽量写全，不要只写 `level`。

### 12.2 version 与 version_code

- `version`：给人看的版本号，如 `1.1.0`
- `version_code`：给程序比较大小用的数字版本号，如 `110`

### 12.3 module 与 category

- `module`：业务模块，如 kana、vocabulary、grammar
- `category`：分类维度，如题型、难度、内容类别

### 12.4 score 与 accuracy

- `score`：综合得分
- `accuracy`：正确率

这两个在打字和游戏场景里不要混着用。

## 十三、推荐统一叫法

为了减少混乱，建议后续文档统一使用下面这些写法：

- “AI 对话”，不要一会儿写 AI 聊天，一会儿写对话系统
- “打字竞速”，不要一会儿写 Typing Race，一会儿写打字比赛
- “推箱子记语法”，不要一会儿写推箱子语法，一会儿写语法推箱子
- “学习进度”，对应 `learning_progress`
- “题库”，对应 `question-bank`
- “访问令牌 / 刷新令牌”，分别对应 Access Token / Refresh Token
- “下次复习时间”，对应 `next_review`

## 十四、后续维护建议

这份术语表建议在下面情况同步更新：

- 新增模块
- 新增关键字段
- 新增游戏模式
- 新增 AI 场景
- 文档里出现新的高频概念

如果后续某个词开始频繁出现在多个文档里，就应该把它补进来。
