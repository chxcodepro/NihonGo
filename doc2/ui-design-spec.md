# UI/UX 设计规范

## 一、设计理念

**核心关键词**: 简洁、专注、愉悦

- **简洁**: 信息层次清晰，不堆砌功能入口
- **专注**: 学习时减少干扰，沉浸式体验
- **愉悦**: 微交互增加趣味，成就感驱动学习

## 七、组件规范

### 7.1 按钮

| 变体 | 用途 | 样式 |
|------|------|------|
| Primary | 主要操作 | 实心樱花粉背景 |
| Secondary | 次要操作 | 灰色背景 |
| Ghost | 图标按钮/轻操作 | 透明背景，悬停变色 |
| Outline | 边框按钮 | 边框+透明 |
| Destructive | 危险操作 | 红色 |

尺寸: `sm` (32px高) / `default` (40px) / `lg` (48px) / `icon` (40x40)

### 7.2 卡片

```

│  ┌───┐                  │
│  │ あ │  ← 大字日文      │
│  └───┘                  │
│                         │
│  读音: a                 │
```

## 八、动效规范

### 8.1 过渡时长

```
fast:    150ms  → 悬停、聚焦
normal:  250ms  → 展开、收起
slow:    350ms  → 页面切换、弹窗
spring:  type: spring, stiffness: 300, damping: 30  → 弹性效果
```

### 8.2 动效场景

| 场景 | 动效 | 参数 |
|------|------|------|
| 页面切换 | 淡入+上移 | opacity 0→1, y 20→0, 300ms |
| 卡片翻转 | Y轴旋转 | rotateY 0→180, 400ms |
| 正确反馈 | 缩放+变绿 | scale 1→1.1→1, 200ms |
| 错误反馈 | 抖动+变红 | x 0→-10→10→0, 300ms |
| 列表加载 | 依次淡入 | 每项延迟50ms |
| 进度条 | 宽度动画 | width, spring |
| Toast通知 | 右侧滑入 | x 100→0, 200ms |
| 成就解锁 | 放大+粒子 | scale 0→1.2→1 + confetti |

### 8.3 打字练习动效

```
正确输入: 字符变绿，微缩放 (scale 1.05)
错误输入: 字符变红，抖动 (shake 5px)
光标: 闪烁竖线 (opacity 0↔1, 530ms)
完成: 全屏烟花/撒花 (canvas-confetti)
```

## 十、无障碍 (A11y)

1. **对比度**: 文字对比度 >= 4.5:1 (WCAG AA)
2. **键盘导航**: 所有交互元素可Tab聚焦
3. **ARIA标签**: 图标按钮必须有 `aria-label`
4. **语义化**: 使用正确的HTML标签 (nav, main, section, article)
5. **焦点指示**: 聚焦元素有可见的焦点环 (ring)
6. **字体缩放**: 支持浏览器字体缩放至200%
7. **动画偏好**: 尊重 `prefers-reduced-motion`
